# 🤖 Автономний AI Sales Агент (Prospecting -> OpenAI Outreach -> React Dashboard)

Це технічне завдання (ТЗ) збережено як перспективний проект для портфоліо та майбутньої реалізації.

---

## 🎯 Суть проекту:
Створення універсальної автономної системи (AI Sales Agent) для автоматичного пошуку потенційних клієнтів (проспектингу), персоналізованої email-рассылки на основі аналізу сайту компанії та розумної класифікації вхідних відповідей (з автоматичним припиненням кампанії при реакції ліда). Керування всіма процесами здійснюється через кастомний веб-інтерфейс (десктопний дашборд).

Перша робоча ніша: платформа будівельної техніки в Німеччині та ЄС ([prostobau.de](https://prostobau.de/)), але архітектура системи розробляється повністю ніше- та мовно-незалежною.

---

## 🛠️ Запропонована архітектура та стек:
* **Платформа автоматизації:** n8n (модульна структура з незалежними воркфлоу).
* **AI-Ядро:** OpenAI API (GPT-4o / GPT-5-mini / OpenRouter).
* **База даних:** Supabase (хмарна PostgreSQL для синхронізації n8n та фронтенду в реальному часі).
* **Фронтенд (Дашборд):** React + Vite (Vanilla CSS, sleek dark mode з неоновими акцентами).
* **Канали зв'язку:** SMTP (вихідні листи через виділений домен-сателіт), IMAP (читання відповідей), Telegram Bot API (щоденні звіти).

## 📋 Повне ТЗ від замовника:

### 1. Пошук компаній (Prospecting)
* Автоматичний щоденний збір ~100 нових компаній за гнучкими запитами через Google Maps (Apify).
* Повна кастомізація без програміста: зміна ніші, регіону, країни, мови та лімітів.
* **Захист від дублів:** Перевірка компанії перед обробкою за 5 полями (домен, email, телефон, Google Maps URL, назва).

### 2. Обогащення та Аналіз сайтів
* Сканування сайту знайденої компанії.
* Визначення послуг, товарів, географії роботи, контактів, імені власника.
* Створення короткого бізнес-опису компанії через OpenAI.
* Пріоритет збору email: `персональний робочий` -> `sales` -> `vertrieb` -> `kontakt` -> `info`. Якщо email відсутній — компанія пропускається.

### 3. Генерація та Валідація листів
* Генерація повністю індивідуального листа через OpenAI (без шаблонів, враховуючи специфіку сайту).
* Стиль: коротко, професійно, дружньо, без тиску, до 120 слів.
* **Валідація:** Автоматична перевірка готового листа перед відправкою (контроль на наявність вигаданих фактів, довжину та рівень персоналізації). При помилці — перегенерація (до 3 спроб).

### 4. Відправка пошти (Outbound)
* Використання окремого домену-сателіту та окремих скриньок для холодного аутрічу (захист репутації головного домену).
* Обов’язкове налаштування SPF, DKIM, DMARC та запуск автоматичного прогріву пошти (Instantly/Warmy).
* Поступова плавна відправка листів протягом робочого дня з випадковими завершеннями.
* **Follow-up:** Рівно один лист через 5 днів, якщо немає відповіді, спаму чи bounce.

### 5. Обробка відповідей (Inbox Management)
* Автоматичний аналіз вхідних відповідей через ШІ.
* Класифікація: `зацікавлений`, `не зацікавлений`, `opt-out (більше не писати)`, `автовідповідь (відпустка)`, `потрібна ручна відповідь`.
* При отриманні будь-якої відповіді (або помилки відправки) розсилка для цього ліда негайно і назавжди припиняється.

### 6. Desktop Dashboard (Панель керування)
* **Головна:** статус роботи агента, статистика (знайдено, оброблено, відправлено, відповіді, зацікавлені, помилки).
* **Таблиця лідів:** фільтрація, пошук та перегляд історії.
* **Керування запитами:** форма додавання/редагування/видалення пошукових фраз, лімітів, пріоритетів та регіонів.
* **Налаштування:** зміна моделей OpenAI, поштових скриньок, лімітів відправки, термінів follow-up, мови та ніші без зміни коду n8n.
* **Системні логи:** детальний опис дій та помилок.
* **Emergency STOP:** Велика кнопка **STOP** для миттєвої зупинки всіх процесів (пошук, генерація, відправка, follow-up) та кнопка швидкого перезапуску.
* **Режими роботи:** *Test Mode* (все виконується, але листи не відправляються — для перевірки якості) та *Auto Mode* (повністю автономний запуск).
