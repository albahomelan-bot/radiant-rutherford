# Архитектурное предложение: Система авто-лидогенерации для агентства недвижимости

Предлагается два варианта архитектурного решения для автоматизации прогрева собственников недвижимости с помощью ИИ (OpenAI), n8n и WhatsApp. 

---

## ВАРИАНТ 1. Официальный WhatsApp Business Cloud API + OpenAI Assistants API (Максимальная стабильность)

Ориентирован на «белые» методы работы, исключающие риски быстрой блокировки номера, и использование официального API от Meta.

### 1. Архитектура и сервисы
* **Триггер и логика:** n8n (Cloud или Self-hosted).
* **Канал связи:** Официальный **WhatsApp Business API (WABA)** напрямую от Meta Cloud API или через провайдера (360dialog / Twilio).
* **Интеллект и Память:** **OpenAI Assistants API** (модель `gpt-4o-mini`). Управление памятью переписки делегировано OpenAI через механизм `Threads`. n8n сохраняет только `Thread ID` в Google Sheets и не хранит историю сообщений локально.
* **CRM:** Google Sheets (подключение через официальный n8н-ноду).

### 2. Workflow (Процессы)
1. **Инициация:** Менеджер добавляет строку в Google Sheets ➔ n8n раз в 5 минут проверяет новые строки ➔ Создает уникальный `Thread ID` в OpenAI Assistants и записывает его в строку ➔ Отправляет клиенту первое сообщение в WhatsApp.
   * *Важно:* Первое сообщение должно быть **шаблоном (Template Message)**, одобренным Meta.
2. **Получение ответа:** Клиент отвечает ➔ n8n ловит входящий Webhook от WhatsApp ➔ Находит по номеру `Thread ID` в Google Sheets ➔ Передает входящее сообщение в этот Thread ➔ Запускает раунд генерации ассистента (`Run Run`).
3. **Генерация ответа ИИ:** OpenAI Assistant считывает системный промпт (инструкцию риелтора), данные объекта из Google Sheets (чтобы не переспрашивать их) и историю `Thread`. Формирует ответ.
4. **Отправка:** n8n забирает ответ ИИ ➔ Отправляет клиенту в WhatsApp ➔ Записывает текст сообщения и обновляет статус/этап разговора в Google Sheets.

### 3. Плюсы и Минусы
* **Плюсы:**
  * **Минимальный риск бана:** Номер привязан официально, Meta разрешает бизнес-переписку.
  * **Простое управление памятью:** OpenAI сам хранит и кэширует контекст всей переписки в Thread.
  * **Надежность доставки:** Официальные серверы Meta работают без сбоев.
* **Минусы:**
  * **Шаблонные ограничения:** Первое сообщение *обязано* быть согласованным шаблоном. Инициализировать диалог свободным текстом нельзя.
  * **Стоимость:** Meta берет плату за каждый открытый диалог (диалоговые сессии платные, около $0.06 - $0.08 за инициацию переписки бизнесом).

## ВАРИАНТ 2. Неофициальный WhatsApp API (Green-API / Z-API) + n8n Chat Model c базой PostgreSQL (Максимальная гибкость и экономия)

Ориентирован на обход жестких правил Meta, возможность писать первыми любой текст (без согласования шаблонов) и полный контроль над расходами.

### 1. Архитектура и сервисы
* **Триггер и логика:** n8n.
* **Канал связи:** Неофициальный шлюз **Green-API** или **Z-API** (работают через эмуляцию WhatsApp Web на реальном номере).
* **Интеллект и Память:** OpenAI Chat Model (`gpt-4o-mini`) + нода **Postgres Chat Memory** в n8n. История сообщений для памяти ИИ сохраняется в базе данных **PostgreSQL**.
* **CRM:** Google Sheets.

### 2. Workflow (Процессы)
1. **Инициация:** Новая строка в Google Sheets ➔ n8n сразу отправляет *произвольное, «живое» стартовое сообщение* через Green-API (например: *«Здравствуйте, Марк! Увидел ваше объявление о продаже квартиры...»*). Создает запись о сессии в локальной БД PostgreSQL.
2. **Диалог:** Клиент отвечает ➔ Входящий Webhook от Green-API прилетает в n8n ➔ n8n обращается к PostgreSQL, достает последние 10 сообщений переписки (буфер памяти) ➔ Передает их в OpenAI Chat Model вместе с промптом риелтора и характеристиками квартиры ➔ ИИ генерирует ответ ➔ n8n отправляет его через Green-API ➔ Сохраняет ответ в Postgres и обновляет статус в Google Sheets.

### 3. Плюсы и Минусы
* **Плюсы:**
  * **Свобода переписки:** Можно писать любой текст первым сообщением, вставлять эмодзи, форматировать как угодно без модерации Meta.
  * **Низкая цена:** Нет платы за сессии Meta. Оплачивается фиксированная подписка на Green-API (около $15–$20/мес за безлимит).
  * **Контроль контекста:** Локальное хранение истории в PostgreSQL позволяет гибко чистить память или обрезать слишком старые сообщения для экономии токенов.
* **Минусы:**
  * **Риск блокировок:** Если собственники будут часто нажимать кнопку «Спам», WhatsApp быстро заблокирует номер (нужен прогрев аккаунтов и ротация номеров).
  * **Сложность инфраструктуры:** Нужно разворачивать и поддерживать базу данных PostgreSQL для истории переписки.

## Масштабируемость решений

Поведение систем при увеличении объемов контактов:

| Объем контактов | Вариант 1 (WABA + OpenAI Assistants) | Вариант 2 (Green-API + Postgres Memory) |
| :--- | :--- | :--- |
| **100 контактов** | Идеально. Нагрузка незаметна. Лимиты Google Sheets не достигаются. | Идеально. Легко настраивается, минимальные затраты. |
| **1 000 контактов** | Работает стабильно. Скорость WABA высокая. Нагрузка на OpenAI кэшируется. | Работает хорошо. PostgreSQL легко справляется с сессиями. Требуется следить за банами номера WhatsApp. |
| **10 000 контактов** | **Вузкое горлышко — Google Sheets API**. При частых обращениях Google Sheets начнет выдавать ошибку лимитов (Rate Limit 100 запросов в минуту). **Решение:** перейти с Google Sheets на полноценную базу данных (PostgreSQL) в качестве CRM. | **Google Sheets начнет зависать**. Необходимо полностью мигрировать CRM-часть на PostgreSQL или подключить KeyCRM/AmoCRM по API. Потребуется распределение трафика на 3-5 разных номеров WhatsApp для избежания банов. |

| Статья расходов | Вариант 1 (Официальный WABA) | Вариант 2 (Неофициальный Green-API) |
| :--- | :--- | :--- |
| **Хостинг n8n** | $20/мес (n8n Cloud) или $5/мес (VPS) | $20/мес (n8n Cloud) или $10/мес (VPS + Postgres) |
| **WhatsApp API** | ~ $0.07 за диалог ➔ **$70/мес** (оплата сессий Meta) | Фиксированная подписка ➔ **$15/мес** (безлимит) |
| **OpenAI API** | ~$0.005 за длинный диалог (gpt-4o-mini) ➔ **~$10 - $15/мес** | ~$0.003 за диалог (контролируемый буфер) ➔ **~$8 - $12/мес** |
| **Итого в месяц:** | **~$100 - $105 / мес** | **~$33 - $42 / мес** |

## Потенциальные риски и методы борьбы с ними

1. **Блокировка WhatsApp (Особенно актуально для Варианта 2):**
   * *Риск:* Номер банят за спам.
   * *Решение:* 
     * Использовать только «прогретые» номера (аккаунты, с которых уже велась переписка).
     * Первое сообщение должно быть максимально мягким и вовлекающим, не содержать прямых продаж (например: *«Здравствуйте! Нашел ваше объявление по квартире в районе Х. Она еще продается?»*).
     * Использовать рандомизацию задержек перед ответами бота (имитация ввода текста человеком).
2. **«Галлюцинации» ИИ и выдумка фактов:**
   * *Риск:* ИИ может назвать неверную цену или соврать о характеристиках объекта.
   * *Решение:* В системном промпте жестко прописать: *«Используй ТОЛЬКО данные из переданного JSON-объекта. Если ИИ спрашивают то, чего нет в данных (например, высоту потолков), отвечай: 'Не подскажу навскидку, надо заглянуть в документы, чуть позже напишу вам'»*.
3. **Потеря памяти переписки:**
   * *Риск:* Бот забывает, о чем общались вчера.
   * *Решение:* В Варианте 1 это решено на стороне OpenAI Threads. В Варианте 2 лимит памяти в Postgres настраивается на последние 10-15 сообщений, чего полностью достаточно для удержания контекста сделки.
4. **Дублирование сообщений:**
   * *Риск:* Из-за задержек Webhook отсылается дважды, ИИ отвечает два раза на одно сообщение.
   * *Решение:* n8n должен проверять ID входящего сообщения (`message_id` от WhatsApp). В базе данных Postgres или Redis ставится временный замок на обработку этого `message_id` на 5 секунд.
