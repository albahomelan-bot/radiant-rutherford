Multi-Format Resume Parser

Automated Document Parsing & Structured Data Extraction Автоматичний аналіз документів та вилучення даних

An n8n-powered background service that watches incoming Gmail messages, extracts candidate resumes from DOCX, PDF, or image attachments, parses contacts and experience fields using AI, and outputs clean structured data.

Фоновий сервіс на базі n8n, який відстежує вхідну пошту Gmail, витягує резюме кандидатів із вкладень форматів DOCX, PDF або зображень, розпізнає контакти та досвід роботи за допомогою ІИ та зберігає структуровані дані.

The Problem (HR Friction) Проблема (Рутина рекрутера)

HR specialists and recruiters waste hours downloading dozens of resume attachments from emails, scanning them manually, copying applicant info (names, phone numbers, emails, skills), and typing them one by one into spreadsheet trackers or CRM dashboards.

Рекрутери витрачають години на скачування десятків резюме з пошти, ручний перегляд файлів та копіювання контактної інформації (імена, телефони, email, навички) для занесення кандидатів до таблиць обліку або CRM-систем.

The Solution (Automatic Parser) Рішення (Автоматичний парсер)

An n8n automation flow triggered by new applicant emails. It downloads attachments, checks formats, converts document pages into images to extract text via OCR if needed, parses key profile fields using structured LLM schemas, and saves them in clean JSON format.

Сценарій n8n автоматично спрацьовує на нові листи. Він завантажує вкладення, розпізнає формати, конвертує сторінки в зображення для OCR-розпізнавання (за потреби), структурує профілі кандидатів через мовну модель та виводить готовий JSON.

System Flow Крок за кроком

Document Parsing Architecture Схема роботи парсера

1

Email Monitoring Trigger Тригер моніторингу пошти

The n8n Gmail node acts as a listener, capturing new messages containing matching keywords (e.g., "Resume", "CV", "Job Application") and downloads attachments. Нода Gmail в n8n виступає слухачем, відловлюючи нові листи за ключовими словами ("резюме", "відгук") та завантажуючи вкладені файли.

2

Format Routing & Conversion Форматування та конвертація

Binary objects are evaluated. PDFs and DOCX are parsed, while scanned images are routed to the OCR engine for text extraction. Бінарні об'єкти проходять валідацію. PDF та DOCX аналізуються напряму, а скановані зображення надсилаються на OCR-вузол для вилучення тексту.

3

AI Schema Extraction Вилучення даних ШІ-моделлю

GPT-4o-mini receives the raw text and extracts structured information (Name, Email, Phone, Skills, Work Experience, Education) using JSON schema validation. GPT-4o-mini аналізує текст резюме та витягує структуровані поля (ім'я, email, телефон, навички, досвід роботи, освіта) згідно з заданою схемою JSON.

4

Structured Export (Google Drive / Sheets) Структурований експорт даних

The parsed applicant data is sent directly to Google Sheets database and the original file is archived on Google Drive in organized subfolders. Сформований профіль кандидата записується в Google Sheets, а оригінальний файл резюме переноситься на Google Drive у відповідну папку.
